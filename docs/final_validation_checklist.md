# Final Validation Checklist

Use Swagger UI as the primary validation interface:

```text
http://localhost:8000/docs
```

## 1. Docker Startup

Run:

```bash
docker compose up --build
```

Expected:

```text
api, worker, postgres, and redis containers start successfully
api runs alembic upgrade head
worker logs Gemini configuration status
```

## 2. Swagger Access

Open:

```text
http://localhost:8000/docs
```

Expected:

```text
Swagger UI loads
health and jobs endpoints are visible
endpoint descriptions and response examples are visible
```

## 3. CSV Upload

In Swagger:

```text
POST /jobs/upload
```

Upload:

```text
transactions.csv
```

Expected:

```json
{
  "job_id": "uuid",
  "status": "pending"
}
```

## 4. Job Processing

In Swagger:

```text
GET /jobs
GET /jobs/{job_id}/status
```

Expected:

```text
status moves from pending to processing to completed
row_count_raw=95
row_count_clean=85
error_message=null
```

## 5. Transaction Persistence

In Swagger:

```text
GET /jobs/{job_id}/results
```

Expected:

```text
transactions array is populated
dates are ISO formatted
amounts are numeric
currency values are INR or USD
status values are SUCCESS, FAILED, or PENDING
missing categories become Uncategorised or Gemini-classified values
```

## 6. Anomaly Detection

Expected:

```text
anomalies array is populated when rules match
is_anomaly=true
anomaly_reason is HIGH_AMOUNT_ACCOUNT_MEDIAN or USD_DOMESTIC_MERCHANT
```

## 7. Gemini Classification

Expected with valid `GEMINI_API_KEY`:

```text
rows with original_category=null have final_category updated to an allowed category
llm_failed=false on successfully classified rows
worker logs rows sent to Gemini, parsed categories, and updated row count
```

Allowed categories:

```text
Food, Shopping, Travel, Transport, Utilities, Cash Withdrawal, Entertainment, Other
```

## 8. Summary Generation

Expected:

```text
summary exists
summary.narrative is generated by Gemini
summary.risk_level is low, medium, or high
top_merchants is populated
category_breakdown is populated
anomaly_count matches anomalies
```

## 9. Results Endpoint

Expected response shape:

```json
{
  "job": {},
  "summary": {},
  "anomalies": [],
  "category_breakdown": {},
  "transactions": []
}
```
